# EasyChart V0.8 유동성 소유 목표 연구 설계

작성일: 2026-07-23 KST  
상태: 사전등록·실행 전

## 1. 문제 정의

V0.7 SR Flip+FVG의 다음 M5 시가 arm은 거래 31건과 80.65% 승률을 만들었지만 합계 손익은 -1.090R이었다. 목표 종류별로는 pivot이 양수, 반대 FVG가 큰 음수였으며, 모든 활성 반대 FVG·OB를 독립 terminal target으로 취급한 규칙이 경제적으로 과도했다.

V0.8은 장면·손절·진입을 바꾸지 않는다. C봉 마감에 확정된 정보만으로 **목표 유동성의 소유권**과 **H1/H4 방향 허용**만 다시 고정한다.

## 2. 사전등록 가설

주가설은 `PIVOT_ONLY + NOT_OPPOSED`다.

1. terminal target은 진행 방향의 확정·미소비 M15/H1/H4 pivot만 허용한다.
2. LONG은 당시 H1 또는 H4가 DOWN이면 거부하고, SHORT은 H1 또는 H4가 UP이면 거부한다.
3. V0.7 경계·FVG·formation stop·다음 M5 실제 시가 진입은 그대로 유지한다.
4. 목표와 문맥은 V0.7 C봉 마감 시점에 실제로 알려진 값만 사용한다.

절제 실험은 다음 네 가지뿐이다.

- pivot only, 문맥 없음
- pivot only, H1/H4 not-opposed
- pivot only, strict H1 delivery
- 동급 이상 반대 OB/FVG 안에 위치한 pivot + not-opposed

V0.7의 모든 목표 허용 arm과 기존 연구 선두를 기준선으로 함께 재생한다.

## 3. 데이터 분리

### 개발 자료

기존 V0.3~V0.7에서 이미 본 여섯 개 14일 구간이다. 이 자료는 오염됐으며 OOS로 재명명하지 않는다.

### 잠금 홀드아웃

실행 결과를 보기 전에 고정한 네 개 기간에서 BTC·ETH를 동시에 사용한다.

- 2024-03-01 ~ 2024-03-15
- 2025-03-01 ~ 2025-03-15
- 2025-10-01 ~ 2025-10-15
- 2026-05-01 ~ 2026-05-15

총 112 종목일, 56 포트폴리오 운용일이다. 각 기간에 BTC와 ETH가 겹치므로 전역 단일 pending/open 슬롯 경쟁을 실제로 재생한다. 기간은 고정 달력 앵커로 선택했으며 전략 결과로 고르지 않았다. 각 평가 시작 전 28일은 H1/H4·피벗·구역 feature warm-up으로만 사용하고, 거래 후보와 성과 분모에는 포함하지 않는다.

## 4. 데이터 무결성

평가 창과 28일 warm-up을 포함하는 공식 Binance USD-M 월별 5분봉 archive와 각 `.CHECKSUM`을 내려받는다. SHA-256 불일치, 중복 timestamp, 5분봉 누락, 비정상 OHLCV가 하나라도 있으면 연구 실행을 실패시킨다. 원본 archive URL·hash와 정규화 CSV hash는 manifest에 남긴다.

## 5. 체결·비용·위험

- 신호는 M15 C봉 마감에 확정된다.
- 다음 M5 시가가 실제 진입가이며 그 가격으로 수량을 다시 계산한다.
- stop·target 동시 접촉 등 기존 보수적 replay 규칙을 유지한다.
- BTC·ETH 합산 pending/open 최대 한 개다.
- 기준 비용, 1.5배 비용, 2배 비용을 비교한다.
- 위험률은 3%로 고정하며 이번 실험에서 최적화하지 않는다.
- 자동매매계좌:은행계좌 비율과 레버리지는 최적화하지 않는다.

## 6. 자동 판정

주가설이 연구 선두 후보가 되려면 신규 홀드아웃에서 다음을 모두 만족해야 한다.

- 단독 arm 기준 비용 순 R 양수
- 단독 arm 1.5배 비용 순 R 양수
- 기존 선두와 결합했을 때 기준 비용 로그성장이 선두 단독보다 큼
- 1.5배 비용에서도 결합 로그성장이 선두 단독보다 큼
- 결합 최대낙폭 증가가 2%p 이하

최종 목표 게이트는 위 조건에 더해 다음을 요구한다.

- 완료 거래 수가 포트폴리오 운용일 이상
- 비용 차감 후 기하 일평균 수익률 1% 이상

홀드아웃 결과를 본 뒤 같은 기간에서 규칙·threshold·arm을 재선택하지 않는다. 실패하면 결과를 보존하고 별도 가설과 새 잠금 기간으로 넘어간다.
